Overview

The Term Project was a group assignment to design and implement an ETL pipeline and the underlying data warehouse to accommodate a Big Data application. We formed teams of 5 and worked with one or more selected Datasets. We defined a business use case around those datasets and engineered a data processing system that incorporated atleast 3 technologies that was studied in class.

We thought Big for this project by thinking about data volume, data sources, velocity, and variety. The application we built is able to process data from multiple sources, handle large volumes of data, and process data in real-time. What we came up with was an application that holds critical data that needs to be protected and processed in a secure environment.

Business Use Case

Our team came up with a application that can solve the emergency response crisis in New York City. When we did our research we found that response times to emergency situations are getting longer and longer. We thought that this crisis can be solved through medical professionals responding to both emergencies and non-emergencies on our platform called MedConnect. MedConnect is a platform that will give users the ability to order a medical professional to their location, like Uber but for medical situations, especially in non-emergencies. We thought that by doing this it caan reduce the wait time in the hospital's emergency centers thus freeing up space for actual emergencies.

MedConnect would have the option to also consult with doctors via the platform, order prescriptions, and order medical supplies (future feature). It would be a central hub to help you in times of emergencies. Think about this, not knowing how to resuscitate someone in real-time but MedConnect being there to help walk you through it with simulations and eventually a medical professional will be on live and on their way to help in the situation. This can be a game changer for emergency situations where most of us are not trained professionals but we can be more prepared.

MedConnect will know your health history and can automatically prescribe you the right medication in emergency situations so the medical professionals can be able to properly assist with needed medicines if they need to pick it up at a pharmacy. The approvals would already be built in with doctors orders for the scripts to be sent rapidly. Emergency responders would be able to have your health information already through the app so they know what to expect and can be prepared for the situation. MedConnect will be equipped with real-time data to help users know how long they may have in certain situations until the patient will start to be in a critical condition. As mentioned before with the resuscitation, time is everything when trying to resuscitate someone and seeing an estimate to how long it can be before the patient is in a critical condition can be proactive.

Tech Stack

Apache Spark

Fast, scalable data processing for batch and streaming. Supports Python, SQL, and Machine Learning.

PostgreSQL

A secure relational database for healthcare data that can support JSON type format like medical records.

MongoDB

NoSQL document base storage for patient data that requires flexible schema and rapid iteration.

Redis

In-memory cache for sessions and real-time management of data. Cuts query latency.

Apache Kafka

Real-time streaming for hospital, clinic, and IoT data. Built for high throughput.

AWS S3

Durable cloud storage for data lakes. Low cost archives and backups for compliant healthcare data.
