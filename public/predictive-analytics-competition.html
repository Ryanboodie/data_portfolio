Credit Card Spending

Overview

A credit card company has provided historical data on its active customers. The data includes demographic and behavioral information on customers.
Your task is to build a model to predict monthly credit card spend for individual customers.

Description


Feature Description

  • customer_id: unique identifier
  • age: Age of the customers in years
  • gender: Gender of the customer (Male/Female)
  • marital_status: Marital Status (Single/Married)
  • num_children: Number of children in the household
  • education_level: Highest Educational level (high school, bachelors, masters, etc.)
  • region: Customer's home region (south, midwest, northeast, west)
  • employment_status: Employment Status (employed, self-employed, unemployed, student)
  • owns_home: Whether the customer owns their home (1 = yes, 0 = no)
  • has_auto_loan: Whether the customer has an auto loan (1 = yes, 0 = no)
  • annual_income: Annual income (in dollars $)
  • credit_score: Credit score of customers (300-850)
  • num_credit_cards: Number of credit cards
  • credit_limit: Credit card limit (in dollars)
  • tenure: Number of years the customer held these credit cards
  • card_type: (gold, standard, platinum)
  • num_transactions: Number of credit card transactions in the month
  • avg_transactions_value: Average dollar value of those transactions
  • online_shopping_freq: Number of online purchases made per month
  • reward_points_balance: Total reward points accumulated by customer
  • travel_frequency: Number of trips taken in past month
  • utility_payment_count: Number of utility payments made with card
  • monthly_spending (Target variable): Total monthly spending on the credit card in dollars

    • Evaluation


    Goal

    Build a predictive model to estimate monthly credit card spending of individual customers based on a rich set of customer attributes, including demographics, credit behavior, transaction activity, and lifestyle indicators.


    Metric

    You must train and validate predictive models that estimate monthly_spend. Submissions will be evaluated based on RMSE (root mean squared error) on the scoring data. Scoring data is split into a public and private dataset. Performance on the public dataset will be posted on Public Leaderboard which will be visible during the competition. Score on the private dataset will be shared on the Private Leaderboard at the conclusion of the competition. Your performance will be judged based on your rank on the Private Leaderboard. Lower your RMSE, higher your rank. Kaggle allows you to select the submission to use for Private Leaderboard. Unless you are extremely confident in the virtues of a particular submission, it is best to allow Kaggle to automatically use your top Public Leaderboard submission.


    Submission File

    The submission file should be in text format (.csv) with only two columns, customer_id and monthly_spend. The monthly_spend column must contain predicted monthly_spend. Number of decimal places to use is up to you. The file should contain a header and have the following format: 

     
                
                    customer_id,monthly_spend
                        20451981,1594.2299893341963
                        23251656,2134.613263862634
                        64082260,1909.1365812010806
                        82505923,1498.4460331618411
                        89981994,2092.6787820448426

    An example submission file (example_submission.csv) is shared with the set of files under Data.


    Sample Code


    Here is an illustration in Python of how you can create a model, apply it to scoring_data.csv to prepare a submission file. 

     
                 
                import pandas as pd
                    data = pd.read_csv('analysis_data.csv')
                    X = data.loc[:,['credit_score', 'travel_frequency']]
                    y = data.monthly_spend

                    from sklearn.linear_model import LinearRegression
                    reg = LinearRegression()
                    reg.fit(X,y)
                    reg.predict(X)

                    scoring_data = pd.read_csv('scoring_data.csv')
                    scoring_data_X = scoring_data.loc[:,['credit_score', 'travel_frequency']]
                    pred = reg.predict(scoring_data_X)

                    submission_file = pd.DataFrame({'customer_id': scoring_data.customer_id, 'monthly_spend': pred})
                    submission_file.to_csv('submission_file_1.csv',index = False)

    Summary

    In this project, I used Python and it's related libraries (OLS, Scikit-learn, Numpy, Pandas, Matplotlib, Seaborn) to create multiple machine learning models to predict monthly credit card spending of individual customers based on a rich set of customer attributes, including demographics, credit behavior, transaction activity, and lifestyle indicators.

    Since this was a numerical outcome I focused mostly on linear regression as a way to get the most accurate predictions. I built models from simple to advance such as linear regression, polynomial regression, decision trees, random forests and gradient boosting. In my Code Reports below you'll see I combined two of those models using a Voting Regressor to get the best possible predictions.

    This assignment made me more intrigued by machine learning as I was able to apply what I've learned in class to a real-world problem. I was also able to see the impact of different models on the predictions and how they can be combined to get the best possible results. I can take this knowledge I have gained and apply it to other similar problems that businesses face. This challenge made me more curious as a Data Scientist and what I learned most was to ask more thorough questions as I written my code. I also learned the importance of preliminary research such as preprocessing, data mining, and exploratory data analysis to get a better understanding of the problem and the dataset.
