Written Report

Keeping it simple

For most of this competition, I focused on one core approach: linear regression using Ordinary Least Squares (OLS). I chose it because linear regression is one of the easiest models to interpret and explain to stakeholders. The more complex a model becomes, the harder it is to translate those insights into business decisions. Early on, I also learned that not every variable deserves a place in the model. Through feature selection, I discovered that several predictors added more noise than value, and removing them helped create a cleaner story around monthly_spend.

Model 5

In this stage, I ran a quick analysis to see which variables might relate to monthly spending. I started by checking whether there was any clear linear trend. For example, when I plotted credit_limit against monthly_spend, the scatter plot showed a lot of noise, high variance, and a low R value. In other words, credit_limit alone didnt explain spending very well.

However, when I included more predictors in Model 5, the model began to take shape. The RMSE dropped, the R improved, and the combined predictors gave me a more convincing narrative about what influences spending. While not every coefficient appeared meaningful, the model produced stronger performance and a clearer view of the relationships in the dataset.

Statistical Significance The OLS summary report became one of my most valuable tools. The p-values helped me understand which variables had a meaningful relationship with monthly spending. I used a threshold of 0.05 to guide my decisions. If a predictor fell below that threshold, it signaled that it was statistically significant and likely worth including in future versions of the model. This step pushed me to think more critically about what belonged in the model rather than letting the algorithm decide for me.

Missteps along the way

Looking back, I can clearly see one missed opportunity: I should have spent more time exploring and transforming the data before modeling. A more thorough Exploratory Data Analysis (EDA) would have helped me uncover patterns, identify outliers, and think about what transformations or feature engineering might reduce the RMSE.

If I had created more visualizationsbar charts, histograms, scatter plots, and relationship chartsI could have seen interactions between variables more clearly. I also could have built helper functions to normalize features, create ratios, and generate binary flags. These kinds of engineered features might have strengthened the model and improved prediction accuracy.

When I later tested an ensemble model using Random Forest and Linear Regression through soft voting, I achieved an RMSE of about 178, my best score so far. Unfortunately, I could not get the score to register on Kaggle, possibly due to a submission or preprocessing error. Still, it showed the potential of blending models when the data is properly prepared.

Learning outcomes This competition taught me that strong analysis does not begin with modelingit begins with understanding the data. Much of data work is repetitive, but what separates a strong analyst is the ability to reshape data into something more meaningful. Tools like feature selection helped me identify which predictors mattered most, but I also learned that domain knowledge can make those outcomes even more insightful.

I also learned that outliers matter. Some models can overfit or distort predictions when extreme values are present, and while I wondered whether polynomial features might smooth that issue, it ultimately did not provide a better fit.

Overall, this challenge blended theory and experimentation. Reducing the RMSE wasnt just about applying modelsit was about exploring, questioning assumptions, and sometimes testing ideas even when they might not work. That mindset is at the core of being a good analyst, engineer, or developer: curiosity, patience, and a willingness to think differently.

#Reviewing Model1 summary

# To get model summary-like information, you can calculate metrics like R-squared and coefficients
from sklearn.metrics import r2_score, mean_squared_error, root_mean_squared_error
import numpy as np

#Calculate RMSE
rmse = np.sqrt(np.mean(np.square(reg.predict(X) - y)))
print(f"Rmse: {rmse}")

# Calculate R-squared
r2 = r2_score(y, reg.predict(X))
print(f"R-squared: {r2}")

# Print the coefficients
print(f"Coefficients: {reg.coef_}")

# Print the intercept
print(f"Intercept: {reg.intercept_}")
